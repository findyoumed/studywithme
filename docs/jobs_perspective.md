> "단순함은 궁극의 정교함이다" - 스티브 잡스

## 🎭 잡스의 핵심 철학

### 1. Focus (집중)

**"사람들은 집중이 '예스'라고 말하는 것이라 생각하지만, 집중은 '노'라고 말하는 것이다"**

### 2. Simplicity (단순함)

**"단순함은 복잡함보다 어렵다. 단순하게 만들기 위해서는 생각을 정리해야 한다"**

### 3. User Experience (경험)

**"기술로 시작하지 마라. 고객 경험에서 시작해 기술로 돌아가라"**

### ❌ 잡스가 거절했을 것들

#### 1. "너무 많은 것을 보여주고 있어"

```
문제:
- 동시에 2개의 영상 (YouTube + 카메라)
- 여러 버튼들 (Copy, Live, Settings)
- 기술적 용어 (room-w7ntzo8)
- 중요하지 않은 정보들

잡스의 질문:
"사용자가 정말 원하는 것은 무엇인가?"
→ 답: 방해받지 않고 집중하고 싶다
```

#### 2. "설명이 필요한 UI는 실패한 UI야"

```
문제:
- "Go Live" 버튼이 무엇인지 직관적이지 않음
- room-w7ntzo8이 무슨 의미인지 모름
- SCORE가 무엇을 측정하는지 불명확
- 얼굴 인식 포인트가 왜 있는지 설명 필요

잡스의 질문:
"할머니가 사용할 수 있나?"
```

#### 3. "매직이 없어"

```
문제:
- 기술적으로는 작동하지만 감동이 없음
- 단순한 타이머와 다를 바 없음
- 사용자가 "와!" 할 순간이 없음

잡스의 질문:
"이걸 친구에게 자랑하고 싶을까?"
```

## 🎨 잡스라면 이렇게 만들었을 것

### Phase 1: 핵심만 남기기 (The First Principle)

#### 사용자 스토리

```
"나는 집중해서 공부하고 싶다.
 내가 얼마나 잘 집중하고 있는지 알고 싶다.
 그게 전부다."

→ 그렇다면 다른 모든 것은 필요없다
```

```
┌─────────────────────────────────────┐

│                                     │

│                                     │

설명:
- 열면 바로 이 화면
- 버튼 하나만 있음
- 클릭하면 마법이 시작됨
```

## 💎 잡스가 집착했을 디테일

### 1. Typography (타이포그래피)

```css
/* 현재 */
font-family: "Inter", sans-serif;
font-weight: 400;

/* 잡스식 */
font-family: "SF Pro Display", system-ui; /* Apple 서체 */
font-weight: 600;
letter-spacing: -0.03em; /* 타이트한 자간 */
line-height: 1.2; /* 빡빡한 줄간격 */
```

### 2. Animation (애니메이션)

```javascript
// 현재: 갑작스러운 변화
element.style.display = "block";

// 잡스식: 부드러운 전환
element.animate(
  {
    opacity: [0, 1],
    transform: ["scale(0.95)", "scale(1)"],
  },
  {
    duration: 400,
    easing: "cubic-bezier(0.4, 0, 0.2, 1)", // Apple의 ease-out
  },
);
```

### 3. Haptic Feedback (햅틱)

```javascript
// 집중 시작/종료 시
navigator.vibrate([10, 5, 10]); // 애플 스타일 섬세한 진동

// 마일스톤 달성 시
navigator.vibrate([50, 30, 50, 30, 100]); // 축하 패턴
```
