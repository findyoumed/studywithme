### 핵심 원칙

**1. Focus (집중)** - 중요한 것만 남기고 나머지는 제거

- ❌ 50-50 분할 화면 → ✅ 타이머 하나에 집중
- ❌ 여러 버튼들 → ✅ 버튼 하나만

**2. Simplicity (단순함)** - 설명이 필요없는 UI

- ❌ "room-w7ntzo8" → ✅ "홍길동의 집중실"
- ❌ 기술 용어 → ✅ 일상 언어

**3. Magic (마법)** - "와!" 하는 순간

- ❌ 갑작스러운 변화 → ✅ 부드러운 애니메이션
- ❌ 단순 알림 → ✅ 감동적인 피드백

**잡스가 만들었다면:**

```
화면의 80%는 비어있음
타이머가 주인공
모든 것이 직관적
매 순간이 아름다움
```

### 1. ⭐ **SCORE 표시 개선** (우선순위: 높음)

#### 현재 문제

- SCORE가 너무 작고 눈에 잘 안 띔
- 검은 배경에 흰 글씨로만 표시되어 시각적 임팩트가 약함
- 사용자가 가장 중요하게 봐야 하는 정보인데 주목도가 낮음

#### 개선 방안

```
현재: SCORE: 00:00:00 (작은 흰 글씨)

개선안 1 (카드 형태):
┌─────────────────┐
│ ⏱️  집중 시간    │
│ 00:00:00        │  ← 크고 굵은 폰트
│ 🏅 Gold Master  │  ← 현재 등급
└─────────────────┘

개선안 2 (프로그레스 바):
집중 점수: 00:00:00  [████████░░] 80%
```

**구체적 수정:**

- 폰트 크기: 24px → 32px
- 배경: 반투명 카드 추가 (glassmorphism)
- 색상: 그라데이션 적용 (#4ade80 → #22c55e)
- 아이콘 추가: ⏱️ 또는 🎯

### 2. 🎥 **Go Live 버튼 상태 표시** (우선순위: 높음)

#### 현재 문제

- 버튼이 켜졌는지 꺼졌는지 한눈에 파악 어려움
- 아이콘만 있고 텍스트 레이블이 없음
- 클릭했을 때 피드백이 약함

#### 개선 방안

**방법 1: 텍스트 레이블 추가**

```
현재: [📡]

개선: [📡 Go Live]  (꺼짐 상태)
      [⏹️ Stop]     (켜짐 상태)
```

**방법 2: 색상 + 애니메이션**

```css
/* 꺼짐 상태 */
.live-btn-icon {
  background: rgba(255, 255, 255, 0.1);
  color: white;
}

/* 켜짐 상태 */
.live-btn-icon.active {
  background: linear-gradient(135deg, #ef4444, #dc2626);
  color: white;
  animation: pulse 2s infinite;
}

@keyframes pulse {
  0%,
  100% {
    box-shadow: 0 0 0 0 rgba(239, 68, 68, 0.7);
  }
  50% {
    box-shadow: 0 0 0 10px rgba(239, 68, 68, 0);
  }
}
```

**방법 3: "LIVE" 표시 강조**

```
현재: [LIVE] 작은 빨간 박스

개선:
┌─────────────┐
│ ● LIVE      │  ← 깜빡이는 빨간 점
│ 3 viewers   │  ← 시청자 수
└─────────────┘
```

### 3. 🏷️ **Room 이름 개선** (우선순위: 중간)

```
- "홍길동's Focus Session"
```

```
- "프로그래밍 집중실"
```

**방법 3: 이름 편집 기능**

```
[room-w7ntzo8 ✏️]  ← 클릭하면 편집 가능
```

### 4. 📹 **카메라 영역 개선** (우선순위: 중간)

#### 현재 문제

- 카메라가 너무 어둡고 흐릿함
- 얼굴 인식 포인트(분홍색 점)가 산만할 수 있음
- 카메라 투명도 조절이 직관적이지 않음

#### 개선 방안

**방법 1: 밝기 자동 조정**

```javascript
// 저조도 환경 감지 및 자동 밝기 조절
if (averageBrightness < threshold) {
  canvas.style.filter = "brightness(1.3) contrast(1.1)";
}
```

**방법 2: 얼굴 인식 포인트 옵션화**

```
설정 메뉴에 추가:
☑ 얼굴 인식 포인트 표시
☐ 집중도 영역만 표시
☐ 포인트 숨김
```

**방법 3: 카메라 컨트롤 개선**

```
현재: 투명도 슬라이더만 존재

개선:
┌─────────────────┐
│ 카메라 설정     │
│ 밝기:  [●━━━━] │
│ 투명도: [━━━●━] │

└─────────────────┘
```

### 6. ♿ **접근성 개선** (우선순위: 중간)

#### 현재 문제

- 버튼에 텍스트 레이블이 없어 스크린 리더 사용 어려움
- 색맹 사용자를 위한 대안 표시 부족
- 키보드 단축키 없음

#### 개선 방안

**방법 1: ARIA 레이블 추가**

```html
<!-- 현재 -->
<button id="btnGoLive">
  <span class="material-symbols-outlined">podcasts</span>
</button>

<!-- 개선 -->
<button id="btnGoLive" aria-label="라이브 방송 시작" aria-pressed="false">
  <span class="material-symbols-outlined">podcasts</span>
  <span class="sr-only">라이브 방송 시작</span>
</button>
```

**방법 2: 키보드 단축키**

```
Ctrl + L : 라이브 시작/중지
Ctrl + C : 룸 링크 복사
Space    : 비디오 재생/일시정지
Ctrl + M : 카메라 켜기/끄기
```

**방법 3: 색맹 대응**

```
LIVE 표시:
- 색상만: ❌ (빨간색만 의존)
- 개선: ✅ "● LIVE" (점 + 텍스트)
```

### 7. 🎨 **시각적 계층 구조** (우선순위: 낮음)

#### 현재 문제

- 모든 요소가 거의 비슷한 크기와 색상
- 중요도에 따른 시각적 구분이 약함
- 다크 테마만 있어서 밝은 환경에서 눈이 피로함

#### 개선 방안

**방법 1: 크기와 색상 계층화**

```
1순위 (가장 중요): SCORE, 현재 영상
  → 크고 밝은 색

2순위 (중요): Live 버튼, Room 정보
  → 중간 크기, 강조 색상

3순위 (보조): Settings, Copy 버튼
  → 작고 중성 색상
```

**방법 2: 라이트 모드 추가**

```css
/* 다크 모드 (기본) */
:root {
  --bg-primary: #0f172a;
  --text-primary: #ffffff;
}

/* 라이트 모드 */
:root[data-theme="light"] {
  --bg-primary: #f8fafc;
  --text-primary: #0f172a;
}
```

**방법 3: 카드 기반 디자인**

```
┌─────────────┐ ┌─────────────┐
│ SCORE 카드  │ │ ROOM 카드   │
│             │ │             │
└─────────────┘ └─────────────┘
┌─────────────────────────────┐
│      유튜브 영상 카드       │
│                             │
└─────────────────────────────┘
```
